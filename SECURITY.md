# Security Policy

## Supported Versions

Only the latest release on the `master` branch is supported.

## Reporting a Vulnerability

Report security issues privately to **poczta@bbasztura.eu**.

Do not open a public GitHub issue for security vulnerabilities.

Include:
- A description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The affected version or commit.

We aim to respond within 72 hours and to provide a fix or mitigation within 14 days of confirmation.

## Responsible Disclosure

We ask that you:
- Give us a reasonable amount of time to fix the issue before public disclosure.
- Do not access or modify other users' data.
- Do not degrade service availability.

## Scope

This policy covers the Homebox MCP server code in this repository. It does not cover vulnerabilities in Homebox itself, in the MCP protocol specification, or in third-party dependencies (report those to their respective maintainers).
